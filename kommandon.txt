$ git init 
//initierar git, när du skapat upp 
ett projekt och vill lägga in det i github använder du det

$git add <file> lägger till filen till index
git add . //lägger till alla filer
git add *.html //lägger till alla html-filer

git rm --cached [index.html] tar bort filen från repot


$git status 
//kollar statusen för trädet.
Känner av om det finns filer i git-mappen där det finns ändringar
som inte blivit commitade


$git commit// commitar ändringarna
När du commitar
skriv git commit
wim editorn dyker upp-> tryck i för att kunna skriva i den
Kommentera bort initial commit
tryck escape för att komma ur skrivläget
skriv :wq för att genomföra commiten. 

Du kan även skriva in detta (enklare)
git commit -m 'another change'


$git push// pushar ditt lokala repositary till github
$git pull// när du ska hämta senaste versionen från github.
$git clone// clonar ett externt repository till din lokala dator


Github laddas ned härifrån: 
https://git-scm.com/download/win

När du laddat ned git kan du köra den här i git bash för att testa att det fungerar
git --version
fungerar även från vanliga cmd

skapa filer i cmd
touch index.html

skapa branch
git bransch login (login kan bytas mot något annat)
sedan går du över till den branschen såhär (då får det inte ligga kvar filer som inte är addade): 
Switched to branch 'login'

sedan går du tillbaka till master såhär: 
git checkout master
(då ser du att katalogen ändras när du går in i utforskaren)

därefter kör du git merge [branschens namn] för att merga

för att koppla till externt repository
$ git remote

sedan går du till repositoriet som du har skapat, exempel detta
(commandon dyker upp på github när du skapat repositorium)
git remote add origin https://github.com/stefansahlin/githubtest.git

för att pusha dit kör du detta:
git push -u origin master
Därefter får du skriva in användarnamn och lösenord (skriv 
då in användarnamn och inte mailadress)
